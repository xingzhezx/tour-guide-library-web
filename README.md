# 导游资料库（手机 App 版 / PWA）

这是一个可安装到手机桌面的导游资料库，支持：

- 导游讲解词管理
- 景区介绍管理
- 注意事项清单
- 录音文件上传与播放
- 音频自动转文案并新建景区
- TXT 文案自动识别并新建景区
- PDF 文章导入（自动提取文本并填充字段）
- 扫描件 PDF 导入（OCR 文字识别）
- 本地自动保存 + JSON 导入导出
- 景区详情目录导航（点击目录项可定位与预览）

## 手机优化点

- 小屏幕下改为底部双标签：`景区列表` / `编辑详情`
- 按钮和输入区域都做了触控尺寸优化
- 顶部工具栏吸顶，单手操作更稳定

## 统一导入说明

1. 点顶部 `导入数据`，可一次选择多个文件。
2. 支持类型：`JSON`、`TXT`、`DOS文本(.dos)`、`PDF`、`Word(.doc/.docx)`、常见音频（mp3/m4a/wav 等）。
3. 系统会自动识别类型并执行：
- `JSON`：单文件导入时按完整数据恢复；与其他文件混合导入时按“新增景区”方式导入。
- `TXT`：自动识别“景区介绍/讲解词/注意事项”，并自动新增景区。
- `DOS文本`：自动识别常见老编码（GBK/GB18030 等），减少乱码。
- `PDF`：自动提取文本；扫描件自动 OCR（中文+英文），并自动新增景区。
- `Word`：支持 `.docx/.doc` 自动提取文本并新增景区。
- `音频`：自动语音转写与文案优化，并自动新增景区。
- 导入后的识别结果会自动精修（口头语清理、段落重排、注意事项规范化）。

提示：
- 首次使用 PDF / OCR / 音频转写需要联网下载模型或解析库。
- 首次使用 `.docx` 导入需要联网加载 Word 解析库（mammoth）。
- OCR 默认最多识别前 12 页；超出会在导入结果中提示。
- `.doc` 为旧格式，解析成功率受文件编码和内容结构影响。
- 音频文件不做大小限制，文件越大处理时间越长。
- 导入后的音频会保存在景区中，并支持点击条目自动播放。
- 录音文件优先保存到 IndexedDB，避免 localStorage 空间不足。

## 安装为手机 App（推荐）

### Android（Chrome）

1. 把本目录部署到一个 HTTPS 地址（如 GitHub Pages/Netlify/Vercel）。
2. 手机 Chrome 打开网址。
3. 点击菜单里的 `添加到主屏幕` 或 `安装应用`。
4. 安装后会像独立 App 一样打开（无浏览器地址栏）。

### iPhone（Safari）

1. 手机 Safari 打开同一个 HTTPS 地址。
2. 点底部 `分享`。
3. 选择 `添加到主屏幕`。

## 本地电脑调试

1. 进入目录：`/Users/xingzhe/Desktop/游戏开发/tour_guide_library`
2. 启动静态服务：`python3 -m http.server 8080`
3. 浏览器访问：`http://localhost:8080`
4. 如果升级后手机仍是旧页面，请关闭 App 后重新打开，或在浏览器里刷新 1-2 次让缓存更新。

## 文件结构

## 注意

- 音频文件较大时，浏览器处理和保存会更慢。
- 导出 JSON 默认不包含音频二进制（音频保存在浏览器 IndexedDB）。
- 清理浏览器数据会导致本地记录丢失，请定期导出 JSON 备份。
